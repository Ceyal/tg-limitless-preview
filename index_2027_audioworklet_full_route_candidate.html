    5. Timer math (future S3 / S4 — not implemented here)

    Target duration source: independent session target (seconds integer or decimal TBD) stored only in volatile session state until persistence slice; must never mirror #rec_duration_input.

    Elapsed / remaining: monotonic wall-safe basis using a primitive chosen during S3 code review (not declared in this document). Display strings rounded to whole seconds for coarse UI; sub-second precision optional behind a flag.

    Rounding: floor for remaining when counting down to zero; elapsed may ceil or floor consistently (pick one in S3 and document).

    Completion: transition Running→Completed when remaining hits zero; freeze displays; no auto-loop unless explicit later slice.

    Paused elapsed: accumulate active segments only; pause freezes both consumption and remaining countdown.

    Tab visibility: timer may drift when backgrounded; spec prefers monotonic reference, not catching up burst audio/export.

    Drift: acceptable display skew vs wall clock; never compensate by coupling to audio callbacks.

    Accessibility: state changes should use aria-live="polite" on a dedicated status node; avoid rapid chatter.

    6. Export boundary

    #rec_duration_input remains the sole MediaRecorder duration source. Session target must not override, autofill, mirror, mutate, or read export duration unless a later signed slice says otherwise. No auto-start or auto-stop export from session state alone.
